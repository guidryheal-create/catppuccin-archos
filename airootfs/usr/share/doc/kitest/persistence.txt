Kitest OS - persistent live overlay (USB)
==========================================

This uses mkinitcpio-archiso’s **cow_label** option (see upstream boot parameter
docs). It is **not** the Calamares installer: you still get **SDDM -> Plasma**
like the default live entry; the only difference is where the overlay is stored.

If you see **“failed to mount root device”** / **“can’t access TTY”**, that is an
**initramfs** issue (ISO / squashfs / persistence), **not** desktop theming. On the
live ISO read **`/usr/share/doc/kitest/initramfs-troubleshooting.txt`**; in the
source tree see **`docs/troubleshooting-initramfs.md`**.

Boot menu names (match systemd-boot / Syslinux; ASCII hyphens for BIOS compatibility)
-------------------------------------------------------------------------------------

  * Kitest OS - live session (..., BIOS / UEFI)
    Default RAM overlay; install with **Calamares** after login from the desktop.

  * Kitest OS - live session (..., BIOS / UEFI) with speech
    Same, with speakup.

  * Kitest OS - persistent live (USB, ..., BIOS / UEFI)
    Overlay on a partition whose **filesystem label** is **KITEST_PERSIST**
    (underscores, not hyphens).

Quick setup (USB)
-----------------

1. Write the ISO to the stick (first partition is the ISO / FAT volume).

2. Add a second partition on the same drive (e.g. ext4).

3. Set the filesystem label **exactly** (underscore between words):

     KITEST_PERSIST

   Wrong: `KITEST-PERSIST` -- that is a **different** label; the kernel will not
   match **cow_label=KITEST_PERSIST**.

4. Under Linux the device link looks like:

     /dev/disk/by-label/KITEST_PERSIST

   (If your tools show a hyphen, you used the wrong label.)

5. Boot **"Kitest OS - persistent live (USB, ...)"** in the firmware menu.

6. Session changes are stored on that partition (overlay path follows archiso
   rules, typically under a `persistent_<ISO volume label>/`-style path on
   that device).

Notes
-----

- **Calamares** is started from the live desktop (or autostart), not from
  choosing "persistent" in the boot menu.
- Persistence needs a **writable** partition; DVD / read-only media cannot
  store the overlay.
- **QEMU / Proxmox / libvirt:** same rule -- attach a **second** virtual disk,
  format it with **mkfs.ext4 -L KITEST_PERSIST** (from the host or another VM),
  then boot **persistent live**. Otherwise use **live session** (see `README.md`).
- **"Failed to mount root" / overlayfs errors:** boot **live session** first.
  If the extra disk is wrong format or label, **cow_label** can fail.
- **"Failed to mount root" / initramfs:** see **initramfs-troubleshooting.txt** in
  this directory.
- Tune RAM overlay size with **cow_spacesize=** (see Arch Wiki / archiso docs).
